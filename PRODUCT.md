# Product

## Register

product

## Users

Sparkly Tails staff — a small team, likely just one or two people. They open this app from inside the Shopify admin (it's an embedded app, launched via the Apps section) to set up and manage tiered/volume pricing on products. Sessions are infrequent and task-focused: create a tier group, assign products, check the resulting prices, go live. Not a daily-driver dashboard someone stares at all day — it's opened when pricing needs to change, then closed.

## Product Purpose

Lets staff define volume pricing tiers (e.g. "5+ units → 15% off, 10+ units → 18% off") and reconcile them into real Shopify automatic discounts, computed from each product's actual live price — never a placeholder or an example. Success looks like: a staff member can create a tier group, assign products, see the exact resulting price per product before committing, and trust that "Go live" makes Shopify's real discounts match what's on screen — including staying in sync automatically as products are added or removed from a live group.

## Brand Personality

Should feel native to Shopify admin — not bolted on as a third-party tool. The bar is Shopify's own Polaris design language: clean, restrained, confident, information-dense where it needs to be (pricing tables), spacious where it doesn't. Comparable in spirit to best-in-class internal tools (Linear, Stripe dashboard): clarity and trustworthiness over decoration, since this handles real pricing data that affects real discounts.

## Anti-references

No specific named anti-references. Generally: avoid the generic AI-generated SaaS look (gradient text, hero-metric tiles, identical card grids, decorative glassmorphism) — none of that fits an embedded admin tool that's meant to feel like a natural extension of Shopify admin, not a marketing surface.

## Design Principles

- **Feels native to Shopify admin, not bolted on.** Visual language should sit comfortably alongside Polaris — neutral surfaces, restrained color, no decorative flourish that would look out of place inside the Shopify admin chrome.
- **Clarity over decoration.** This is a task tool for staff handling real pricing decisions; every element should serve the task (create a group, assign products, verify prices, go live) rather than embellish it.
- **Show the real number, not a placeholder.** The product's core trust promise — computing actual resulting prices from real Shopify data — should be reflected in the UI: prices and slot counts should always read as concrete, verified facts, never approximations.
- **Task-focused, not dashboard-y.** Sessions are short and infrequent; the UI should support "get in, do the task, get out" rather than encourage lingering or browsing.
- **Trustworthy with money.** Since this reconciles real automatic discounts, the interface should never make the user second-guess whether an action (assign, go live, take offline) had the effect it claims.

## Accessibility & Inclusion

Standard WCAG AA. No specific known user needs beyond that — good contrast, visible focus states, keyboard navigability, and proper form labeling are the baseline, consistent with Shopify's own Polaris accessibility standards.
